# Руководство по дизайну: приложение-конструктор SCORM-тестов

## Подход к дизайну

**Выбранный подход**: дизайн-система — паттерн «продуктивное приложение»

**Основные референсы**:
- Linear (чистый дашборд, быстрые и понятные сценарии)
- Notion (контент-насыщенные формы, чёткая иерархия)
- Google Forms (интерфейс теста без отвлекающих элементов)

**Обоснование**: это утилитарный инструмент для работы, где важны скорость, ясность и «плотные» интерфейсы с данными. Визуальная часть служит функции, а не украшательству.

---

## Ключевые принципы дизайна

1. **Две разные среды**: разные визуальные языки для Автора (power user) и Учащегося (максимальная фокусировка и простота)
2. **Иерархия информации**: чёткие, легко сканируемые макеты для контента с большим количеством данных
3. **Эффективность сценариев**: минимум кликов, максимум ясности
4. **Сначала доступность**: соответствие WCAG AA во всех взаимодействиях

---

## Типографика

**Семейства шрифтов** (Google Fonts через CDN):
- Основной: `Inter` (400, 500, 600, 700) — UI, основной текст, таблицы данных
- Вторичный: `JetBrains Mono` (400, 500) — фрагменты кода, ID, технические данные

**Шкала размеров**:
- Заголовок страницы/герой: `text-4xl font-bold` (36px)
- Заголовок секции: `text-2xl font-semibold` (24px)
- Заголовок карточки: `text-lg font-semibold` (18px)
- Основной текст: `text-base` (16px)
- Мелкий/метаданные: `text-sm` (14px)
- Мини/лейблы: `text-xs` (12px)

**Межстрочный интервал**:
- Заголовки: `leading-tight`
- Основной текст: `leading-relaxed`
- Плотные данные: `leading-normal`

## Система компоновки

**Базовые отступы** (единицы Tailwind):
- Микро: `2, 3` (0.5rem, 0.75rem) — плотные отступы, паддинги кнопок
- Стандарт: `4, 6, 8` (1rem, 1.5rem, 2rem) — карточки, секции, формы
- Макро: `12, 16, 20` (3rem, 4rem, 5rem) — секции страниц, крупные разделители

**Ширины контейнеров**:
- Дашборд Автора: `max-w-7xl` с полноширинными таблицами данных
- Интерфейс Учащегося: `max-w-3xl` (по центру, фокус)
- Формы: `max-w-2xl`
- Модалки: `max-w-lg` до `max-w-4xl` в зависимости от контента

**Сеточные паттерны**:
- Карточки тем/тестов: `grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-6`
- Банк вопросов: список в одну колонку с раскрытием
- Статистика дашборда: `grid grid-cols-2 md:grid-cols-4 gap-4`

## Библиотека компонентов

### Компоненты дашборда Автора

**Навигация**:
- Левый сайдбар (фиксированный, `w-64`) с иконкой + подписью
- Иконки: Heroicons (outline стиль)
- Активное состояние: подчёркнуто индикаторной полосой

**Таблицы данных**:
- Полосатые строки для удобного сканирования
- Липкая (sticky) строка заголовка
- Действия прямо в строке (иконки: редактировать, удалить, дублировать)
- Футер с пагинацией
- Компактная высота строки: `h-12` до `h-14`

**Карточки** (Темы, Тесты):
- На основе бордеров + лёгкое «приподнимание» при hover
- Хедер: заголовок + меню действий (три точки)
- Тело: ключевые метрики (кол-во вопросов, когда редактировали)
- Футер: теги/бейджи статуса
- Паддинги: `p-6`

**Формы**:
- Паттерн «лейбл над полем»
- Поля на всю ширину с понятными фокус-состояниями
- Многошаговые формы с индикатором прогресса
- Валидация прямо в месте ошибки
- Единая высота инпутов: `h-10` до `h-12`

**Конструкторы типов вопросов**:
- **Один/несколько вариантов**: список с кнопками добавить/удалить, индикаторы radio/checkbox
- **Сопоставление**: две колонки + визуализация линий соединения
- **Ранжирование**: перетаскиваемый список с «ручками» (иконка drag из Heroicons)

**Конструктор теста**:
- Две колонки: Доступные темы (слева) + Выбранные темы (справа)
- Паттерн drag-to-add или клик для выбора
- Панель настроек по теме (кол-во вопросов, правила прохода)
- Общие критерии прохода в липком футере

### Компоненты интерфейса Учащегося

**Прохождение теста**:
- Центрированная компоновка в одну колонку (`max-w-3xl`)
- Фиксированная полоса прогресса сверху (`h-2` на всю ширину)
- Карточка вопроса: большие отступы (`p-8`), много «воздуха»
- Номер вопроса + общее количество заметно сверху
- Крупные варианты ответов с понятным состоянием выбора
- Навигация: кнопки «Назад/Далее», «Отправить тест» после завершения
- Минимум отвлечений — скрыть всё несущественное

**Отрисовка вопросов**:
- **Один вариант**: крупные radio (`w-5 h-5`), строки кликабельны на всю ширину
- **Несколько вариантов**: крупные чекбоксы, понятная индикация множественного выбора
- **Сопоставление**: сетка в две колонки с dropdown или drag-соединениями
- **Ранжирование**: нумерованный список с drag-ручками, понятная обратная связь при перестановке

**Страница результатов**:
- Герой-секция: крупный вывод результата (проценты + дробь) и бейдж «сдан/не сдан»
- Карточка общего статуса с заметной визуальной обратной связью
- Разбивка по темам: раскрывающийся аккордеон или сетка карточек
- По каждой теме: название, балл, прогресс-бар, индикатор «сдан/не сдан»
- Для проваленных тем: блок «Рекомендуемые курсы» со стилизованными карточками ссылок
- Кнопки действий: «Пройти снова», «Посмотреть разбор» (если применимо)

### Общие компоненты

**Кнопки**:
- Primary: сплошная заливка, средняя высота (`h-10`), паддинги `px-6`
- Secondary: контур (border)
- Ghost: только текст + фон при hover
- Icon-кнопки: квадрат `w-10 h-10`
- Все кнопки: скругление `rounded-lg`

**Бейджи/теги**:
- Небольшие «пилюли» `rounded-full`
- Высота: `h-6` (плотные паддинги)
- Использование: тип вопроса, статус теста, теги темы

**Модальные окна**:
- Центрированное окно + бэкдроп с блюром
- Макс. ширина по сложности контента
- Хедер: заголовок + иконка закрытия
- Футер: кнопки действий справа
- Плавная анимация входа (scale + fade)

**Пустые состояния**:
- Иконка по центру (крупная, из Heroicons)
- Заголовок + описание
- CTA-кнопка «создать первый …»
- Отступы: `py-16`

## Макеты страниц

### Дашборд Автора — основной макет
- Фиксированный левый сайдбар (`w-64`)
- Верхняя панель с хлебными крошками и меню пользователя (`h-16`)
- Основная зона: заголовок страницы (title + actions) + скроллируемый контент

### Управление темами
- Сетка карточек тем + карточка «Создать тему»
- Клик по карточке → полноэкранный редактор с формой (название, описание, список рекомендуемых курсов)

### Банк вопросов
- Таблица/список с фильтрами
- Сайдбар фильтров: по теме, по типу
- Быстрое добавление вопроса вверху
- Каждая строка вопроса раскрывается для предпросмотра

### Конструктор теста
- Степпер (Темы → Настройка → Проверка → Экспорт)
- Шаг 1: выбор тем (сетка)
- Шаг 2: настройки по темам (drawCount, правила прохода) в табличном виде
- Шаг 3: общие правила прохода + предпросмотр
- Шаг 4: сводка теста + заметная кнопка «Экспорт SCORM»

### Учащийся — список тестов
- Простая сетка карточек доступных тестов
- Карточка: название, описание, число вопросов, примерное время
- Кнопка «Начать тест» (по hover или всегда видна)

### Учащийся — активный тест
- Полноэкранный режим фокусировки (опционально выход к списку)
- Прогресс-бар закреплён сверху
- Один вопрос на экране
- Навигация снизу

### Страница результатов
- Герой: большой круг/дуга с визуализацией балла
- Сетка карточек результатов по темам (2–3 колонки на десктопе)
- Блок рекомендаций внизу (если есть проваленные темы)

## Доступность и взаимодействия

- Все интерактивные элементы: минимум `44x44px` для тач-таргета
- Поля ввода: чёткое focus-кольцо (2px offset)
- Навигация с клавиатуры: видимые focus-состояния везде
- Для скринридеров: ARIA-лейблы на кнопках только с иконкой
- Ошибки: иконка + текст, не только визуальный намёк
- Загрузка: скелетоны или спиннеры + текст статуса

## Анимации

**Использовать умеренно**:
- Лёгкие переходы для hover (`transition-all duration-200`)
- Появление/исчезновение модалок (`duration-300`)
- Обновление прогресс-бара (плавная смена ширины)
- Toast-уведомления с выездом

**Избегать**:
- Анимаций по скроллу
- Декоративных анимаций
- Сложных многошаговых последовательностей

---

## Иконки

**Библиотека**: Heroicons (outline для навигации/действий, solid для статуса/индикаторов)  
**CDN**: подключение через `<script>`  
**Использование**: единый размер 20px или 24px, в основной навигации — всегда с текстовой подписью

---

## Изображения

Приложение в первую очередь про формы и данные, поэтому изображения — не ключевой элемент. Используйте их только для:
- Иллюстраций пустых состояний (простая линейная графика)
- Аватаров пользователей (круг с инициалами по умолчанию)
- Миниатюр рекомендуемых курсов (опционально, маленький превью)

**Хиро-изображения не нужны** — это рабочий инструмент, а не маркетинговая страница.
